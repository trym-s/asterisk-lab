# Architecture

Durable architecture documentation lives here.

Convention: the main entry document is `app-architecture.md` - it describes
the system's components, their boundaries, and how they interact. A skeleton ships with the
template and is filled during harness initialization (brownfield: derived
from the existing code; greenfield: from the planned layout) and expanded
alongside the first spec. Add further documents per
subsystem or contract as the product grows, and list them here.
