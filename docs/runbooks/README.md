# Runbooks

Operational procedures and workflow rules for this repository.

- `plan-rules.md` - rules governing the root `PLANS.md`.
- `spec-rules.md` - rules governing specs under `docs/specs/`.
- `testing.md` - how to run and interpret the project's checks.
- `local-development.md` - local environment setup and daily commands.

Add one runbook per repeatable operational procedure. Keep runbooks short,
imperative, and safe to follow verbatim.
